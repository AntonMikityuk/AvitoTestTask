Сервис назначения ревьюеров для Pull Request’ов

## Стек

*   **Язык:** C# / .NET 8
*   **Фреймворк:** ASP.NET Core Web API
*   **База данных:** PostgreSQL 15
*   **ORM:** Entity Framework Core (Code First)
*   **Инструменты:** Docker, Docker Compose, Swagger

## Инструкция

**Требование:** Установленный Docker Desktop

1.  **Клонирование репозитория:**
    git clone <https://github.com/AntonMikityuk/AvitoTestTask>
    cd <AvitoTestTask>


2.  **Запуск:**
    Находясь в корне проекта (рядом с файлом docker-compose.yaml), выполните:

    docker-compose up --build

3.  **Доступ к API:**
    После успешного старта контейнеров документация Swagger будет доступна по адресу:
    [http://localhost:8080/swagger/index.html](http://localhost:8080/swagger/index.html)

Примечание: Миграции базы данных применяются автоматически при старте приложения

## Принятые решения и допущения
В процессе реализации были приняты следующие решения:

## Создание пользователей
В спецификации API отсутствует отдельный метод для регистрации пользователей
Пользователи создаются или обновляются автоматически при вызове метода создания команды, это исключает возможность существования пользователя без команды

## Логика флага `isActive`
ТЗ гласит: *"Пользователь с isActive = false не должен назначаться на ревью"*.
Флаг учитывается только в момент назначения, если пользователь был назначен ревьюером, а после стал неактивным, он не удаляется из текущих PR, что предотвращает ситуации, когда PR остается без ревьюеров

## Авторство PR
Неактивные пользователи могут быть авторами PR, ограничение касается только назначения их в качестве проверяющих

## Алгоритм случайного выбора
Для выбора случайных ревьюеров используется сортировка по Guid.NewGuid()
Для текущего объема данных (20 - 200 пользователей) это достаточное решение для равномерного распределения

## Идемпотентность Merge
Если метод merge вызывается для PR, который уже имеет статус MERGED, сервис возвращает код 200 (OK) и текущее состояние объекта, не выбрасывая ошибку, что реализует идемпотентность
